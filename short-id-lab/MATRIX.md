## Prediction scorecard

**4 correct / 1 wrong / 1 partial / 1 void** (7 scenarios)

- **S1** — void (steady-state check ran before a prediction was on record)
- **S2** ✓ — concurrent counter dupes, atomic-counter uniqueness, 10⁴/s throughput mechanism
- **S3** ✗ — **the load-bearing miss.** Predicted first collision at ≈ keyspace D (~10⁹/10¹¹/10¹³); actual ≈ √D (~10⁴/10⁵/10⁶). Missed the birthday bound — collisions accrue by pairs (n²/2D), so first collision at ≈1.25·√D
- **S4** ✓ — control: 0 collisions at 100k (formula slip on the probability, corrected)
- **S5** ✓ — TOCTOU under concurrency; UNIQUE constraint moves the race into the write
- **S6** ✓ (c partial) — Snowflake structural uniqueness; seq ceiling is a floor on time, real throughput binds
- **S7** ◐ — guard correct; naive magnitude off ~60× (rollback re-traverses the whole window, not one tick)

**Read of the trend:** predictions were strong on mechanism everywhere (TOCTOU, seq-reset, monotonic-clock dependency all named correctly before running) — the two misses (S3, S7) were both **magnitude/scale** errors from the same root: underestimating how *quantities accumulate*. S3 = collisions accumulate by pairs (√D not D); S7 = a rollback accumulates across a held window (2,972 not 50). Worth a follow-up lab on "quantities that grow faster than the naive linear intuition" (birthday bound, coupon collector, queue backlog under sustained overload).
